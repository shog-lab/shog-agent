# 自优化

当系统检测到本次对话存在优化信号时，你会收到此指令。按以下 SOP 执行。

## 什么时候该优化

- skill 执行失败或步骤不对
- 用户纠正了你的错误行为
- 发现可复用的经验模式（不是一次性的特殊情况）
- 相同问题反复出现

## 可以改什么

- **skills/**：修 SKILL.md 中的步骤、参数、边界条件、前置条件
- **wiki/**：写入新发现的领域知识、用户偏好、项目约定

## 不能碰什么

- **AGENTS.md**：人设变更由主 group 管理，不要自行修改
- **内建 skills**（`/app/skills/`）：只读，不可修改
- **extensions**：不可修改

## 怎么改

1. 先定位：哪个 skill 或知识点出了问题
2. 读取当前内容，确认问题所在
3. 做最小改动——只改有明确证据支持的部分
4. 不做推测性优化（"也许以后会用到"不是改的理由）
5. 改完后简要说明改了什么、为什么改

## 不该做什么

- 不要重构没问题的 skill
- 不要因为一次特殊情况就改通用流程
- 不要删除现有步骤，除非确认它们是错的
- 不要创建新 skill——只改现有的（创建新 skill 是用户的决定）
